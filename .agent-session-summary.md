## Goal
Fix whiteboard writing lag/freeze by replacing ValueNotifier live-drawing state with a Cubit using freezed state with proper equality/hashCode.

## Constraints & Preferences
- The app must launch without crashing (previous record-type approach and static caches caused crashes).
- Use freezed + equatable so state equality and hashCode are handled automatically, reducing unnecessary rebuilds.
- Avoid disk I/O (removed `_saveState()`) and allocations during active drawing.
- The solution must survive multiple strokes and pointer events without freezing.

## Progress
### Done
- Fixed `shouldRepaint` to only return true when relevant properties change.
- Removed `_saveState()` from `updateStrokeAt` and `updateStrokeTransform` (disk I/O on every move was a major cause of lag).
- Added point simplification (2.0 distance threshold) and 2000-point cap per stroke.
- Removed Stopwatch and debugPrint from painter hot path.
- Removed all static caches (causing launch crashes).
- Simplified active stroke drawing to use `drawLine` per segment.
- Removed watchdog timer (was causing crashes).
- Added selection-related fields back to `whiteboard_view.dart`.
- Removed all ValueNotifiers (`_activeStrokeNotifier`, `_activeStrokeRevision`, `_hoverPositionNotifier`) from `whiteboard_view.dart`.
- Created `LiveDrawingCubit` + freezed `LiveDrawingState` with `activeStroke`, `hoverPosition`, `revision`.
- Replaced all `ValueListenableBuilder` layers with `BlocBuilder<LiveDrawingCubit, LiveDrawingState>`.
- Provided `LiveDrawingCubit` via `BlocProvider.value` in the build tree.
- Verified compilation with `dart analyze` — no new issues.

### Next Steps
- Add `buildWhen` to the Live Layer `BlocBuilder<LiveDrawingCubit>` for finer-grained rebuild control (e.g., skip rebuilds when only completed strokes change).
- Test on device: verify no lag/freeze during rapid drawing.
- If freeze persists, consider `RepaintBoundary` with a separate render layer for the live stroke.

## Key Decisions
- Use Cubit + freezed state with proper equality instead of multiple ValueNotifiers, because freezed's auto-generated equality ensures identical states don't trigger rebuilds.
- The `LiveDrawingCubit` is a stateful cubit owned by `_WhiteboardViewState`, shared via `BlocProvider.value`.
- Points remain mutated in place via `_activeStrokePoints`; the cubit emits `notifyChange()` to bump `revision`.
- Freezed generates proper `==` so emitting identical fields (unchanged revision, same activeStroke) prevents unnecessary BlocBuilder rebuilds.

## Critical Context
- `DrawingState` uses `@Freezed(equal: false)` with custom `==` that checks `identical()` first — this is because deep-equality on large lists freezes the UI. For `LiveDrawingState`, default freezed equality is fine since the state is small.
- The cubit has methods: `startStroke()`, `notifyChange()`, `updateHover()`, `endStroke()`, `cancel()`.
- `_activeStrokePoints` is the shared mutable list (also referenced by `state.activeStroke.points`), set in `_handleDrawingStart` and cleared in `_cancelActiveStroke`/`_cleanupDrawing`.
